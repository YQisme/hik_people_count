# 图片命名规范更新说明

## 更新概述

根据用户需求，已将图片保存和命名规范进行了重大更新，现在图片会按设备名称和员工姓名进行更合理的分类存储。

## 主要变更

### 1. 目录结构变更

**旧格式：**
```
D:/Picture/
└── 192.168.1.100/
    ├── 2024_01_15_14_30_25_Major[5]_Minor[75]_AcsEvent.jpg
    └── 2024_01_15_15_45_12_Major[5]_Minor[75]_AcsEvent.jpg
```

**新格式：**
```
D:/Picture/
├── 前门门禁/
│   ├── 张三/
│   │   ├── 张三_20240115_143025.jpg
│   │   └── 张三_20240115_154512.jpg
│   └── 李四/
│       └── 李四_20240115_162033.jpg
└── 后门门禁/
    └── 王五/
        └── 王五_20240115_143025.jpg
```

### 2. 文件命名规范

**旧格式：**
```
{年}_{月}_{日}_{时}_{分}_{秒}_Major[{主类型}]_Minor[{次类型}]_AcsEvent.jpg
```

**新格式：**
```
{员工姓名}_{年}{月}{日}_{时}{分}{秒}.jpg
```

### 3. URL访问格式

**旧格式：**
```
http://localhost:8080/images/{设备IP}/{文件名}
```

**新格式：**
```
http://localhost:8080/images/{设备名称}/{员工姓名}/{文件名}
```

## 技术实现

### 1. 目录创建逻辑
```csharp
// 获取设备名称
string deviceName = GetDeviceName(deviceIP);
if (string.IsNullOrEmpty(deviceName) || deviceName == "未知设备")
{
    deviceName = deviceIP; // 如果设备名称为空，使用IP作为设备名称

// 处理姓名，移除特殊字符，确保文件夹名称合法
string safePersonName = GetSafeFolderName(personName);

{
    safePersonName = "未知员工";
}

// 创建目录结构：D:/Picture/设备名称/员工姓名/
string dirPath = "D:/Picture/" + deviceName + "/" + safePersonName + "/";
```

### 2. 文件命名逻辑
```csharp
// 生成文件名：姓名+时间
string timeStr = string.Format("{0:D4}{1:D2}{2:D2}_{3:D2}{4:D2}{5:D2}",
    struAcsAlarmInfo.struTime.dwYear, struAcsAlarmInfo.struTime.dwMonth, struAcsAlarmInfo.struTime.dwDay,
    struAcsAlarmInfo.struTime.dwHour, struAcsAlarmInfo.struTime.dwMinute, struAcsAlarmInfo.struTime.dwSecond);

string fileName = safePersonName + "_" + timeStr + ".jpg";
```

### 3. 安全文件夹名称处理
```csharp
private string GetSafeFolderName(string name)
{
    if (string.IsNullOrEmpty(name))
    {
        return "未知员工";
    }

    // 移除或替换Windows文件系统不允许的字符
    string safeName = name;
    char[] invalidChars = Path.GetInvalidFileNameChars();
    foreach (char c in invalidChars)
    {
        safeName = safeName.Replace(c, '_');
    }

    // 移除前后空格
    safeName = safeName.Trim();

    // 如果处理后为空，返回默认值
    if (string.IsNullOrEmpty(safeName))
    {
        return "未知员工";
    }

    return safeName;
}
```

## 优势

### 1. 更好的组织结构
- 按设备名称分类，便于管理不同门禁设备
- 按员工姓名分类，便于查找特定员工的记录
- 层次化的目录结构，更加清晰

### 2. 更直观的文件名
- 文件名包含员工姓名，便于识别
- 时间格式更简洁（YYYYMMDD_HHMMSS）
- 移除了技术性的Major/Minor信息

### 3. 更友好的URL
- URL路径更语义化
- 便于理解和记忆
- 支持中文设备名称和员工姓名

## 兼容性说明

### 1. 向后兼容
- 旧的图片文件仍然可以通过原有URL访问
- 新的图片保存逻辑不会影响现有文件

### 2. 特殊字符处理
- 员工姓名中的特殊字符会被替换为下划线
- 确保文件夹名称在Windows文件系统中合法
- 支持中文、英文、数字等字符

### 3. 默认值处理
- 如果设备名称为空，使用设备IP
- 如果员工姓名为空，使用"未知员工"
- 确保系统在各种情况下都能正常工作

## 注意事项

1. **特殊字符处理**：员工姓名中的特殊字符（如 `\ / : * ? " < > |`）会被替换为下划线
2. **中文支持**：完全支持中文设备名称和员工姓名
3. **大小写敏感**：Windows文件系统不区分大小写，但URL可能区分
4. **路径长度限制**：Windows路径长度限制为260字符，建议设备名称和员工姓名不要过长
