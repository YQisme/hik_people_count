# 中文路径访问说明

## 问题背景

在Web服务器中，URL路径包含中文字符时需要进行URL编码，否则可能导致访问失败。本系统已经实现了自动的URL编码和解码处理，确保中文路径能够正常访问。

## 解决方案

### 1. URL编码处理

系统在生成图片URL时会自动对中文路径进行URL编码：

```csharp
// URL编码，确保中文路径能够正确访问
string encodedDeviceName = Uri.EscapeDataString(deviceName);
string encodedPersonName = Uri.EscapeDataString(personName);
string encodedFileName = Uri.EscapeDataString(fileName);
```

### 2. URL解码处理

系统在处理图片请求时会自动对URL进行解码：

```csharp
// URL解码，处理中文路径
string decodedImagePath = Uri.UnescapeDataString(imagePath);
```

## 访问方式

### 1. 直接中文路径访问

可以直接在浏览器中使用中文路径访问：

```
http://localhost:8080/images/前门门禁/张三/张三_20240115_143025.jpg
```

### 2. URL编码路径访问

也可以使用URL编码后的路径访问：

```
http://localhost:8080/images/%E5%89%8D%E9%97%A8%E9%97%A8%E7%A6%81/%E5%BC%A0%E4%B8%89/%E5%BC%A0%E4%B8%89_20240115_143025.jpg
```

## 故障排除

### 1. 404错误
- 检查文件是否确实存在于对应目录
- 确认文件名和路径完全匹配
- 检查文件系统权限

### 2. 乱码问题
- 确保浏览器支持UTF-8编码
- 检查Web服务器的字符编码设置

### 3. 路径解析错误
- 确认URL编码是否正确
- 检查特殊字符是否被正确处理

## 注意事项

1. **浏览器兼容性**：现代浏览器都支持中文URL，但某些旧版本可能有问题
2. **文件系统**：Windows文件系统完全支持中文文件名和路径
3. **URL长度**：编码后的URL会变长，但仍在合理范围内
4. **特殊字符**：除了中文，系统也会正确处理其他特殊字符的编码

## 技术实现

### 1. 编码处理
```csharp
// 在ACSEventService.cs中
string encodedDeviceName = Uri.EscapeDataString(deviceName);
string encodedPersonName = Uri.EscapeDataString(personName);
string encodedFileName = Uri.EscapeDataString(fileName);
```

### 2. 解码处理
```csharp
// 在SimpleWebServer.cs中
string decodedImagePath = Uri.UnescapeDataString(imagePath);
```

### 3. 图片列表页面
```csharp
// 在图片列表页面中生成编码后的URL
string encodedPath = Uri.EscapeDataString(relativePath.Replace('\\', '/')).Replace("%2F", "/");
string imageUrl = "/images/" + encodedPath;
```

通过以上实现，系统完全支持中文路径的访问，用户可以使用中文路径或编码路径两种方式访问图片。
